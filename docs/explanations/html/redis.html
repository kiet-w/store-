Hệ thống Redis Caching

Tài liệu kỹ thuật chi tiết về cơ chế tích hợp và vận hành Cache-Aside trong dự án.

Sơ đồ Hệ Thống (System Data Flow)

1. THÔNG TIN MODULE

MODULE NAME RedisModule
DECORATORS @Global()
FILE PATH src/redis/redis.module.ts
TRÁCH NHIỆM CHÍNH
Khai báo và cung cấp RedisService cho toàn bộ hệ thống. Sử dụng @Global() để tránh việc lặp lại imports tại các module nghiệp vụ.

2. REDIS SERVICE (src/redis/redis.service.ts)

getOrSet(key, fetchFunction, ttl) Cache-Aside Pattern
MỤC TIÊU
Thực hiện mô hình "Nếu có trong cache thì lấy, nếu không thì query DB và lưu lại". Đây là phương thức quan trọng nhất để tối ưu hiệu suất.
INPUT
key: string, fetchFunction: () => Promise<any>
OUTPUT
Promise<any> (Dữ liệu từ Cache hoặc DB)
LOGIC CHI TIẾT
  1. Kiểm tra sự tồn tại của key trong Redis.
  2. Nếu Hit: Log ⚡ và trả về ngay.
  3. Nếu Miss: Log 🐌 và thực thi fetchFunction().
  4. Lưu kết quả mới vào Redis với TTL tương ứng.
graph TD
A[Start] --> B{Check Cache}
B -- Hit --> C[Return Data]
B -- Miss --> D[Fetch from DB]
D --> E[Store in Cache]
E --> C
reset() Maintenance
MỤC TIÊU
Xóa sạch toàn bộ dữ liệu trong Cache. Thường dùng khi deploy hoặc khi dữ liệu bị sai lệch nghiêm trọng.
DEFENSIVE PROGRAMMING
Phương thức này kiểm tra cả clear() và reset() vì tên gọi có thể thay đổi tùy thuộc vào version của cache-manager và store đang sử dụng.
graph LR
A[Reset Call] --> B{Is clear available?}
B -- Yes --> C[Call clear]
B -- No --> D{Is reset available?}
D -- Yes --> E[Call reset]
D -- No --> F[Do nothing]
get(key)
Mô tả
Wrapper cho cacheManager.get. Trả về null nếu không tìm thấy.
set(key, value, ttl)
Mô tả
Lưu dữ liệu vào cache. Mặc định TTL là 30 giây nếu không truyền vào.
del(key)
Mô tả
Xóa một key cụ thể. Thường dùng cho Cache Invalidation khi dữ liệu thay đổi.
