Hệ thống Authentication & Authorization

Module: AuthModule
Base Route: /auth
Cơ chế: JWT Dual-Token (Access & Refresh)
Bảo mật: HttpOnly Cookies, Token Rotation, Bcrypt Hashing

Sơ đồ Hệ Thống (System Data Flow)

1. Controller Handlers

POST /auth/register AuthController.register()
IN: RegisterDto OUT: AuthResponseDto 
graph TD
A[Request Body] --> B{Email Exist?}
B -- Yes --> C[ConflictException]
B -- No --> D[Hash Password]
D --> E[Create User DB]
E --> F[Return Success]

Tiếp nhận thông tin đăng ký mới. Sử dụng bcrypt với salt rounds là 10 để bảo mật mật khẩu.

  • Kiểm tra email trùng lặp qua userService.findByEmail.
  • Sử dụng AuthResponseDto để chuẩn hóa phản hồi.
POST /auth/login AuthController.login()
IN: LoginDto OUT: User + Cookie 
graph TD
A[Login Request] --> B[Find User]
B --> C{Verify Pass}
C -- Fail --> D[401 Unauthorized]
C -- OK --> E[Generate Tokens]
E --> F[Hash Refresh Token]
F --> G[Set HttpOnly Cookies]
G --> H[Return User Info]

Xác thực người dùng và thiết lập phiên làm việc qua Cookie.

  • HttpOnly: Ngăn Javascript truy cập token.
  • SameSite Lax: Cân bằng bảo mật CSRF và UX.
  • Gắn Refresh Token (đã hash) vào Database để quản lý phiên.

2. Services & Logic

AuthService.generateTokens() Private Service
IN: userId, email, role OUT: Access + Refresh 
graph LR
A[Payload] --> B[Sign Access 15m]
A --> C[Sign Refresh 7d]
B --> D[Token Pair]
C --> D

Tạo cặp token JWT. Access token có thời hạn ngắn (15 phút) để giảm thiểu rủi ro khi lộ, Refresh token dài hạn (7 ngày) dùng để cấp mới access token.

AuthGuard.canActivate() Middleware
IN: Cookie OUT: boolean 
graph TD
A[Read Cookie] --> B{Has Token?}
B -- No --> C[401]
B -- Yes --> D[Verify JWT]
D -- Fail --> E[401]
D -- OK --> F[Check User Status]
F -- Inactive --> G[401]
F -- OK --> H[Attach User to Req]

Chốt chặn bảo vệ các route private. Không chỉ kiểm tra token mà còn kiểm tra trạng thái isActive của user trong DB để vô hiệu hóa tài khoản bị ban ngay lập tức.
