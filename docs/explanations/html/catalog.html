Module Catalog - Tài liệu kỹ thuật

Cung cấp API duyệt danh mục và sản phẩm, tối ưu cho các thao tác đọc dữ liệu lớn (Read-heavy).

Module Name: CatalogModule
Base Routes: /catalog/categories, /catalog/products
Files: categories.*, products.*
Dependencies: PrismaService, RedisService
Core Responsibility: Duyệt sản phẩm, phân trang, lọc nâng cao, SEO slug.

Sơ đồ Hệ Thống (System Data Flow)

Phương thức

Products: getProducts
Products: getProductBySlug
Service: findProducts
Service: getProductsCacheKey
Service: buildProductWhere
Categories: findCategories

getProducts()

GET /catalog/products

Lấy danh sách sản phẩm có phân trang và bộ lọc từ URL Query.

graph LR
A["Query Params"] --> B(Controller)
B --> C["Service.findProducts"]
C --> D["paginated Response"]

getProductBySlug()

GET /catalog/products/:slug

Truy vấn chi tiết một sản phẩm dựa trên slug SEO thân thiện.

graph LR
A["Slug Param"] --> B(Controller)
B --> C["Service.findProductBySlug"]
C --> D["success Response"]

Service: findProducts(query)

Internal Service

Logic lõi thực hiện Promise.all để lấy dữ liệu và tổng số bản ghi đồng thời.

graph TD
A[Start] --> B["Generate Cache Key"]
B --> C{Check Redis}
C -- Hit --> D["Return Cached"]
C -- Miss --> E["Promise.all: findMany + count"]
E --> F["Serialize & Cache"]
F --> G["Return Result"]

getProductsCacheKey()

Private Helper

Tạo cache key duy nhất dựa trên tổ hợp các tham số lọc và phân trang.

graph LR
A[Query DTO] --> B[URLSearchParams]
B --> C[Sort Params]
C --> D[Final Key String]

buildProductWhere()

Private Helper

Xây dựng đối tượng truy vấn Prisma, hỗ trợ tìm kiếm không phân biệt hoa thường và lọc giá Decimal.

graph TD
A[DTO] --> B{Has Search?}
B -- Yes --> C[insensitive mode]
A --> D{Has Price?}
D -- Yes --> E[Decimal gte/lte]
C --> F[PrismaWhereInput]
E --> F
