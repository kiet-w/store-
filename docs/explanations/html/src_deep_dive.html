Triết Lý Thiết Kế: Enterprise Modular Architecture

Dự án chọn kiến trúc Modular thay vì Monolith truyền thống để đảm bảo khả năng mở rộng (Scalability) và tính đóng gói (Encapsulation).

  • Khả năng mở rộng: Có thể tách module thành Microservice dễ dàng.
  • Tính đóng gói: Chỉ chia sẻ những gì explicit exports.
  • Dễ bảo trì: Cô lập logic để viết Unit Test chính xác hơn.

RolesGuard Security Layer
TRÁCH NHIỆM
Kiểm tra quyền truy cập của người dùng dựa trên UserRole. Hoạt động bằng cách đọc Metadata từ Decorator @Roles().
LOGIC
  1. Lấy danh sách Roles yêu cầu từ handler/class qua Reflector.
  2. Lấy user từ request object (đã được AuthGuard gán).
  3. So sánh role của user với danh sách yêu cầu.
  4. Ném ForbiddenException nếu không khớp.
