Hệ thống Quản trị (Admin Module)

Module: AdminModule
Quyền hạn: Role-based Access Control (ADMIN only)
Kiến trúc: Controller -> Service -> Repository -> Cache
Tính năng chính: Quản lý danh mục, sản phẩm, đơn hàng và người dùng.

Sơ đồ Hệ Thống (System Data Flow)

1. Danh mục (Categories)

AdminCategoriesService.createCategory() Service Logic
IN: CreateCategoryDto OUT: Category Entity 
graph TD
A[Start] --> B[Check Unique Slug]
B --> C[Create in DB]
C --> D[Invalidate Cache]
D --> E[Log Audit Trail]
E --> F[End]

Quy trình tạo danh mục mới an toàn:

  • Unique Check: Chặn slug trùng lặp trước khi gửi xuống DB.
  • Cache Invalidation: Xóa cache CATEGORIES để menu trang chủ được cập nhật ngay lập tức.
  • Audit Trail: Ghi lại adminUserId thực hiện thao tác.

2. Đơn hàng (Orders)

AdminOrdersService.updateOrderStatus() Service Logic
IN: orderId, newStatus OUT: Updated Order 
graph TD
A[Check Exists] --> B[Get Old Status]
B --> C[Update DB Status]
C --> D[Log Transition]
D --> E[Notify User?]

Quản lý trạng thái đơn hàng (PENDING, SHIPPING, DELIVERED, CANCELLED).

  • Kiểm tra đơn hàng có tồn tại hay không qua Repository.
  • Ghi log chi tiết lịch sử thay đổi trạng thái.

3. Người dùng (Users)

AdminUsersService.toggleUserLock() Security Logic
IN: targetUserId OUT: Status boolean 
graph TD
A[Check targetUserId] --> B{Self Lock?}
B -- Yes --> C[Throw Error]
B -- No --> D[Update isActive DB]
D --> E[Log Admin Action]

Cơ chế khóa/mở khóa tài khoản người dùng.

  • Self-lock Prevention: Admin không thể tự khóa chính mình để tránh mất quyền truy cập hệ thống.
  • Immediate Effect: AuthGuard sẽ chặn user ngay lập tức khi flag isActive thành false.
