Infrastructure Overview (Prisma & Redis)

TỔNG QUAN HẠ TẦNG

Stack: Prisma ORM (PostgreSQL) + Redis (Cache Manager)

Pattern: Global Modules, Singleton Services, Repository Fallback.

Mục tiêu: Tối ưu hóa hiệu năng đọc/ghi và đảm bảo tính nhất quán dữ liệu bền vững.

Sơ đồ Hệ Thống (Infrastructure Data Flow)

1. Global Initialization

Global

Infrastructure Modules

File: src/prisma/prisma.module.ts, src/redis/redis.module.ts

Cả Prisma và Redis Module đều được đánh dấu @Global() để duy trì duy nhất một connection pool.

graph TD A["AppModule"] --> B["PrismaModule @Global"] A --> C["RedisModule @Global"] B --> D["PrismaService Singleton"] C --> E["RedisService Singleton"]

Why Global?

Connection Pool: Ngăn chặn việc tạo quá nhiều client gây cạn kiệt tài nguyên database (PostgreSQL limit).

Ease of Use: Không cần import thủ công tại từng module nghiệp vụ.

2. Redis Cache Service

Redis

getOrSet() Logic

File: src/redis/redis.service.ts

Cơ chế Cache Aside giúp tối ưu hóa tốc độ phản hồi.

graph TD A[Request Data] --> B{In Redis?} B -- Yes --> C[Return Fast] B -- No --> D[Fetch from DB] D --> E[Save to Redis] E --> F[Return Data]

Technical Breakdown

TTL Default: 30,000ms (30 giây).

Reset Logic: Hỗ trợ cả cache.clear() và cache.reset() tùy thuộc vào driver cache-manager đang dùng.
