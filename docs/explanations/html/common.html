Common Module Technical Manual

THÔNG TIN MODULE

Module Name: CommonModule

Base Scope: Toàn hệ thống (Shared Infrastructure)

Core Responsibility: Quản lý lỗi tập trung, chuẩn hóa phản hồi API, kiểm soát phân quyền (RBAC), và các tiện ích dùng chung.

Sơ đồ Hệ Thống (System Data Flow)

1. Exception Filters

Filter

AllExceptionsFilter

File: src/common/filters/all-exceptions.filter.ts

Bộ lọc ngoại lệ toàn cục giúp chuẩn hóa mọi lỗi thành định dạng JSON nhất quán.

graph TD A[Exception Thrown] --> B{Is HttpException?} B -- Yes --> C[Get Status & Msg] B -- No --> D[Status 500 & Generic Msg] C --> E[Log Error Detail] D --> E E --> F[Send JSON Response]

Technical Breakdown

Decorators: @Catch()

Input: exception: unknown, host: ArgumentsHost

Logic:

  • Sử dụng switchToHttp() để truy cập Express objects.
  • Ghi log chi tiết qua Logger('CrisisManagementTeam').
  • Ẩn thông tin lỗi 500 nhạy cảm khỏi client.

2. Interceptors

Interceptor

TransformInterceptor

File: src/common/interceptors/transform.interceptor.ts

Tự động bọc dữ liệu thành công vào cấu trúc { statusCode, message, data }.

graph LR A[Controller Result] --> B{Already Formatted?} B -- Yes --> C[Return Data] B -- No --> D[Wrap in ApiResponse] D --> C

Technical Breakdown

Method: intercept()

Input: ExecutionContext, CallHandler

Logic:

  • Sử dụng RxJS map() để can thiệp luồng dữ liệu.
  • Kiểm tra sự tồn tại của statusCode và message để tránh bọc lặp.

3. Guards (RBAC)

Guard

RolesGuard

File: src/common/guards/roles.guard.ts

Kiểm soát quyền truy cập dựa trên Role được cấu hình qua metadata.

graph TD A[Request] --> B[Read Metadata] B --> C{Roles required?} C -- No --> D[Allow] C -- Yes --> E[Check User Role] E --> F{Match?} F -- Yes --> D F -- No --> G[Throw Forbidden]

Technical Breakdown

Method: canActivate()

Dependencies: Reflector

Logic:

  • Dùng getAllAndOverride để ưu tiên metadata tại method.
  • Yêu cầu thông tin user từ request (thường gán bởi AuthGuard).
