Validation Logic & Decorators

Deep dive into data transformation, validation pipeline, and custom metadata decorators in NestJS.

CORE RESPONSIBILITY Đảm bảo dữ liệu đầu vào (Request Payload/Query) được làm sạch, chuyển đổi đúng kiểu và kiểm chứng tính hợp lệ trước khi đi vào logic nghiệp vụ.
TECHNOLOGIES NestJS ValidationPipe, class-validator, class-transformer, Prisma Enums.

Sơ đồ Hệ Thống (Validation Pipeline)

1. Data Transfer Objects (DTO)

PaginationQueryDto
TRANSFORM & VALIDATE
File:pagination-query.dto.ts
Fields:page, limit, search, category, minPrice, maxPrice, sort
Logic:@Type, @Transform, @IsInt, @Max

Tại sao viết như vậy?

  • @Type(() => Number): Ép kiểu từ Query String (string) sang Number.
  • @Transform: Trim khoảng trắng và convert "" sang undefined.
  • Security: @Max(100) chặn DoS attack qua tham số limit.
graph TD
    A["Raw Query Params"] --> B{Transform Stage}
    B -- "@Type" --> C["Number Conversion"]
    B -- "@Transform" --> D["Trim & Undefined"]
    C --> E{Validation Stage}
    D --> E
    E -- Success --> F["Validated DTO"]
    E -- Fail --> G["BadRequestException"]

2. Custom Decorators

@Roles(...roles: UserRole[])
METADATA
File:roles.decorator.ts
Key:'roles'
Usage:@Roles(UserRole.ADMIN)

Sử dụng SetMetadata:

Lưu trữ danh sách vai trò vào metadata của method hoặc class. RolesGuard sẽ đọc dữ liệu này qua Reflector để thực hiện kiểm tra quyền.

Sử dụng Rest Parameters (...roles) giúp cú pháp gọi linh hoạt hơn. 
graph LR
    A["Developer"] -- "Adds @Roles" --> B["Controller Method"]
    B -- SetMetadata --> C[(Metadata Storage)]
    D[RolesGuard] -- "Reflector.get" --> C
    D -- Compare --> E{Access Decision}

3. Authorization Guards

RolesGuard
PROTECTION
File:roles.guard.ts
Mechanism:CanActivate
Dependency:Reflector

Luồng xử lý:

  1. Lấy requiredRoles từ metadata của handler và class.
  2. Nếu không có role nào được yêu cầu, cho phép qua (true).
  3. Lấy user.role từ request.
  4. Nếu role không khớp, ném ForbiddenException.
graph TD
    A[Request arrives] --> B[Get Metadata]
    B --> C{Roles defined?}
    C -- No --> D[Allow Access]
    C -- Yes --> E[Get User Role from Req]
    E --> F{Role matches?}
    F -- Yes --> D
    F -- No --> G[Throw ForbiddenException]
