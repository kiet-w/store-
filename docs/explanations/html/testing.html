"E2E-First for Enterprise" Strategy Overview

Hạ tầng E2E: createE2EApp

File Path
test/e2e-app.ts
Core Responsibility
Khởi tạo và cấu hình instance NestJS cho môi trường test, đảm bảo đồng bộ với production (main.ts).
Dependencies
AppModule, PrismaService, cookieParser, AllExceptionsFilter
graph TD A["process.env.NODE_ENV='test'"] --> B["Test.createTestingModule"] B --> C["app.createNestApplication"] C --> D["app.use Global Config"] D --> E["app.init"]

Tại sao phải cấu hình thủ công?

NestJS TestingModule không tự động load các global config từ main.ts. Chúng ta phải gắn cookieParser, ValidationPipe, và AllExceptionsFilter để test case không bị "vượt rào".

const app = moduleFixture.createNestApplication();
app.use(cookieParser());
app.useGlobalPipes(new ValidationPipe({...}));

Tại sao Unit Test lại quan trọng?

Mặc dù E2E test bao phủ rộng, Unit test giúp phát hiện nhanh các lỗi logic thuần túy trong vài mili giây mà không cần khởi động Database.

module.overrideGuard(AuthGuard)
  .useValue({ canActivate: () => true });
